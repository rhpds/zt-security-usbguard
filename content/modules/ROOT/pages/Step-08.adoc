## Review & Resources for USBGuard

This lab taught you how to use USBGuard to control which USB devices can connect to your RHEL 10 system. You learned how to move from trusting all USB devices to only allowing devices you approve.

## Module Summary

* Installed USBGuard on RHEL 10
* Created policies to allow or block USB devices
* Learned about USB device rules (vendor IDs, product IDs, interface types)
* Tested blocking USB flash drives while allowing keyboards and mice
* Created allow-lists for specific trusted devices
* Learned about BadUSB attacks and how USBGuard protects against them

## Key Takeaways

**1. Control Your USB Ports**

USBGuard lets you decide which USB devices can work on your computer. Each device must be approved before it can connect. This stops attackers from plugging in malicious devices.

**2. BadUSB Attacks Are Real**

Cheap tools like USB Rubber Ducky and Flipper Zero can pretend to be keyboards and type malicious commands in seconds. Regular antivirus can't stop them because they look like normal keyboards. USBGuard blocks them by checking device identity.

**3. Rules Work Top to Bottom**

USBGuard reads your rules from first to last and uses the first match. Put your "allow" rules for specific devices first, then "block" rules for everything else.

**4. Three Ways to Create Policies**

* **By Device Type**: Block all USB storage but allow keyboards and mice
* **By Device ID**: Only allow your specific keyboard and mouse (most secure)
* **By USB Port**: Only allow devices plugged into certain ports

**5. USBGuard Has Limits**

USBGuard controls what devices connect, but it doesn't:

* Scan for viruses
* Prevent data theft after a device is allowed
* Detect if a trusted device has been hacked

Always use USBGuard with other security tools.

**6. Always Test Your Policies**

* Back up your rules before changing them
* Test new policies carefully
* Make sure you don't accidentally block your keyboard

**7. Know the Bad Guys**

Common attack tools to block:

* USB Rubber Ducky: `1eaf:0030`
* Flipper Zero: `0483:5740`
* O.MG Cable: `d3c0:d34d`

## Final Thoughts

USB ports are often overlooked in security, but they're a major risk. An attacker with a $50 device can bypass expensive security systems in seconds just by plugging into your computer.

**Why USBGuard Matters:**

Instead of trusting all USB devices, USBGuard makes each device prove it's safe. Even if someone has physical access to your computer, they can't just plug in a malicious device.

**How to Use This:**

1. Start by allowing all current devices
2. Watch and log what happens
3. Slowly block unknown devices
4. Create a process for adding new approved devices

**Remember:**

* USBGuard only controls USB connections, not everything
* It works at the system level, making it hard for attackers to bypass
* Combine it with other security tools for best protection
* Some devices (like cloned USB drives) can trick USBGuard
