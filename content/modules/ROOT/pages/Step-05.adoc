# USBGuard Policy Creation and Management

### 5.1 Backup Current Policy

Always backup before making changes:

[source,bash,role="execute",subs=attributes+]
----
sudo cp /etc/usbguard/rules.conf /etc/usbguard/rules.conf.backup
----

### 5.2 Understanding the Daemon Configuration

View the daemon configuration:

[source,bash,role="execute",subs=attributes+]
----
sudo cat /etc/usbguard/usbguard-daemon.conf
----

Key settings to understand:

RuleFile::
Location of rules file

ImplicitPolicyTarget::
What to do with devices that don't match any rule
* `allow` - Allow by default (permissive)
* `block` - Block by default (recommended)
* `reject` - Reject by default (strict)

PresentDevicePolicy::
How to handle devices present when daemon starts
* `allow` - Allow all present devices
* `keep` - Keep current authorization state
* `reject` - Reject all devices
* `apply-policy` - Apply rules to existing devices

### 5.3 Create a Restrictive Policy

Let's create a policy that:

* Allows built-in USB controllers
* Allows keyboards and mice
* Blocks mass storage devices
* Blocks all other devices

USB devices have interface classes that define their type:

Common interface types:

* `03:00:01` - HID Keyboard
* `03:00:02` - HID Mouse
* `08:*:*` - Mass Storage
* `09:00:00` - USB Hub
* `0e:*:*` - Video devices
* `ff:*:*` - Vendor Specific

[source,bash,role="execute",subs=attributes+]
----
sudo tee /etc/usbguard/rules.conf > /dev/null <<'EOF'
allow with-interface 09:00:00
allow with-interface 03:00:*
block with-interface 08:*:*
block
EOF
----

### 5.4 Validate the New Policy

Before applying, validate syntax:

[source,bash,role="execute",subs=attributes+]
----
sudo usbguard read-descriptor /etc/usbguard/rules.conf
----

If no errors, the file is valid.

### 5.5 Restart USBGuard Service

Apply the new policy:

[source,bash,role="execute",subs=attributes+]
----
sudo systemctl restart usbguard.service
----

Check status:

[source,bash,role="execute",subs=attributes+]
----
sudo systemctl status usbguard.service
----

You may have to close out of the status with a "Ctrl+c" .

### 5.6 Test the Policy

Check which devices are currently allowed:

[source,bash,role="execute",subs=attributes+]
----
sudo usbguard list-devices
----

You should see:
- USB controllers and hubs: `allow`
- Keyboards and mice: `allow`
- Mass storage devices: `block`

### 5.7 Test with a USB Device (optional)

If you are doing this online, you will not have a thumb drive to plug in.
If you doing in your own homelab & you have a USB flash drive:

1. Insert the USB drive
2. Check its status:

[source,bash,role="execute",subs=attributes+]
----
sudo usbguard list-devices
----

You should see it blocked (not authorized).

[source,bash,role="execute",subs=attributes+]
----
# Check system messages
sudo dmesg | tail -20
----

The device should be detected but not authorized for use.

### 5.8 Create an Allow-List Policy

Now let's create a strict allow-list policy (only specific devices allowed):

* Allow USB controllers
* Generate and add specific allow rules for your trusted devices
* Copy the rules for your keyboard, mouse, and other trusted devices here
* And we will add an allow id 046d:c52b serial "1234ABCD" name "Logitech USB Receiver"
* Block everything else by default (implicit with ImplicitPolicyTarget=block)

[source,bash,role="execute",subs=attributes+]
----
sudo tee /etc/usbguard/rules.conf > /dev/null <<'EOF'
allow with-interface 09:00:00
allow id 046d:c52b serial "1234ABCD" name "Logitech USB Receiver"
EOF
----

### 5.9 Add Specific Devices to Allow-List

For each trusted device currently connected:

List devices
[source,bash,role="execute",subs=attributes+]
----
sudo usbguard list-devices
----

For each trusted device, append its rule
[source,bash,role="execute",subs=attributes+]
----
sudo usbguard list-devices | grep "allow id" | sudo tee -a /etc/usbguard/rules.conf
----

Review the file:

[source,bash,role="execute",subs=attributes+]
----
sudo cat /etc/usbguard/rules.conf
----

### 5.10 Test the Allow-List Policy

Restart the service:

[source,bash,role="execute",subs=attributes+]
----
sudo systemctl restart usbguard.service
----

Try connecting a new USB device (one not in the allow-list):

[source,bash,role="execute",subs=attributes+]
----
sudo usbguard list-devices
----

It should be blocked.

### 5.11 Policy for Specific Ports

Create a policy that restricts devices based on physical ports:

[source,bash,role="execute",subs=attributes+]
----
sudo tee /etc/usbguard/rules.conf > /dev/null <<'EOF'
# Allow controllers
allow with-interface 09:00:00

# Allow any device on front USB ports (example: ports 1-1 and 1-2)
allow via-port "1-1"
allow via-port "1-2"

# Block devices on all other ports
block via-port "1-*"
EOF
----

**Note**: Port numbers can be found with `sudo usbguard list-devices`
